# 블로그 기능 추가 구현 계획

## 현재 상태 분석

### 기존 사이트 구조
- **플랫폼**: GitHub Pages (정적 호스팅)
- **기술 스택**: 순수 HTML/CSS/JavaScript (Bootstrap 5 기반)
- **템플릿**: iPortfolio (BootstrapMade)
- **현재 기능**: 포트폴리오 섹션, Contact 폼 (비활성화 상태)

### 요구사항 분석
1. **관리자 기능 (본인만)**:
   - 블로그 글 작성
   - 글 편집/삭제

2. **일반 사용자 기능**:
   - 블로그 글 읽기
   - 댓글 작성

## 구현 방안 비교

### 방안 1: 정적 사이트 생성기 (Jekyll/Hugo) + GitHub Pages

**장점**:
- GitHub Pages가 Jekyll을 기본 지원
- 완전 무료
- Markdown으로 글 작성 가능
- Git으로 버전 관리
- 댓글은 외부 서비스(Utterances, Giscus) 연동

**단점**:
- 관리자 인증 개념이 없음 (Git push가 곧 관리자)
- 댓글은 GitHub 계정 필요
- 기존 사이트 구조 대폭 변경 필요
- 학습 곡선 존재

**구현 단계**:
1. Jekyll/Hugo로 사이트 마이그레이션
2. 블로그 레이아웃 구성
3. 기존 포트폴리오 콘텐츠 통합
4. Utterances/Giscus로 댓글 시스템 추가
5. 로컬에서 글 작성 후 Git push로 배포

---

### 방안 2: 순수 정적 방식 (JSON 데이터 + JavaScript 렌더링)

**장점**:
- 기존 사이트 구조 유지
- GitHub Pages 그대로 사용
- 빠른 로딩 속도
- 서버 비용 없음

**단점**:
- 관리자 UI 없음 - JSON 파일 직접 수정 필요
- 댓글은 외부 서비스 의존
- 실시간 인증/권한 관리 불가능

**구현 단계**:
1. 블로그 데이터 구조 설계 (JSON)
2. 블로그 목록 페이지 생성
3. 블로그 상세 페이지 템플릿 생성
4. JavaScript로 동적 렌더링 로직 작성
5. 외부 댓글 시스템 연동 (Disqus/Utterances)
6. 메인 네비게이션에 블로그 섹션 추가

**데이터 구조 예시**:
```json
{
  "posts": [
    {
      "id": 1,
      "title": "첫 번째 블로그 글",
      "date": "2025-12-17",
      "author": "Yoonsuh Jung",
      "excerpt": "글 요약",
      "content": "전체 내용...",
      "tags": ["개발", "AI"]
    }
  ]
}
```

### 방안 3: 헤드리스 CMS + GitHub Pages

**장점**:
- 관리자 UI 제공 (웹 기반 글 작성)
- 권한 관리 가능
- GitHub Pages 계속 사용
- Markdown 에디터 지원

**CMS 옵션**:
- **Netlify CMS / Decap CMS**: 무료, GitHub 연동, 인증 지원
- **Forestry.io**: 사용자 친화적 UI
- **TinaCMS**: 실시간 편집 가능

**단점**:
- 추가 서비스 설정 필요
- CMS 학습 필요
- 일부 서비스는 유료 전환 가능성

**구현 단계**:
1. Decap CMS 설정 (admin 폴더)
2. CMS config.yml 작성
3. GitHub OAuth 앱 설정 (인증용)
4. 블로그 컬렉션 정의
5. 블로그 페이지 HTML/JS 작성
6. 댓글 시스템 연동
7. CMS로 글 작성/관리

---

### 방안 4: 백엔드 서비스 구축 (Node.js/Python + DB + 호스팅)

**장점**:
- 완전한 관리자 인증/권한 관리
- 실시간 댓글 기능
- 사용자 관리 가능
- 확장성 높음

**단점**:
- 서버/DB 호스팅 비용 발생
- 백엔드 개발/유지보수 필요
- GitHub Pages 외 추가 인프라 필요
- 복잡도 대폭 증가

**필요 기술**:
- 백엔드: Express.js/FastAPI
- 데이터베이스: PostgreSQL/MongoDB
- 인증: JWT
- 호스팅: AWS/Heroku/Railway

## 추천 방안 및 근거

### 추천: 방안 2 (순수 정적) 또는 방안 3 (헤드리스 CMS)

#### 방안 2를 추천하는 경우:
- 블로그 업데이트 빈도가 낮음 (월 1-2회)
- JSON 파일 수정에 거부감 없음
- 최소한의 설정으로 빠르게 시작
- 완전 무료 운영

#### 방안 3을 추천하는 경우:
- 웹 UI로 편하게 글 작성하고 싶음
- 관리자 로그인 기능 필요
- Markdown 에디터 사용
- GitHub 직접 수정 불편함

### 방안 1은 추천하지 않는 이유:
- 기존 사이트 전체 재구축 필요
- 현재 HTML 구조와 스타일 모두 변경
- 학습 비용 대비 효용 낮음

### 방안 4는 추천하지 않는 이유:
- 개인 포트폴리오 사이트에 과도한 복잡도
- 서버 비용 및 유지보수 부담
- GitHub Pages 무료 호스팅 장점 상실

## 구체적 구현 계획 (방안 3 - Decap CMS 기준)

### Phase 1: CMS 설정
1. `admin/` 디렉토리 생성
2. `admin/index.html` - CMS 진입점
3. `admin/config.yml` - CMS 설정 파일
4. GitHub OAuth App 등록 (관리자 인증용)

### Phase 2: 블로그 데이터 구조
1. `_posts/` 디렉토리 생성 (Markdown 파일 저장)
2. 포스트 메타데이터 정의 (frontmatter)
3. 포스트 파일명 규칙: `YYYY-MM-DD-title.md`

### Phase 3: 블로그 UI 구현
1. `blog.html` - 블로그 목록 페이지
   - 포스트 카드 레이아웃
   - 페이지네이션
   - 태그/카테고리 필터
2. `blog-post.html` - 블로그 상세 템플릿
   - Markdown → HTML 렌더링 (marked.js)
   - 댓글 섹션
   - 이전/다음 포스트 네비게이션
3. `assets/js/blog.js` - 블로그 로직
   - Markdown 파일 fetch
   - 포스트 렌더링
   - 목록 생성

### Phase 4: 댓글 시스템
**옵션 A - Utterances** (추천):
- GitHub Issues 기반
- 완전 무료
- 오픈소스
- GitHub 계정 필요

**옵션 B - Giscus**:
- GitHub Discussions 기반
- Utterances보다 기능 많음
- 무료

**옵션 C - Disqus**:
- 가장 대중적
- 무료 플랜 광고 있음

### Phase 5: 네비게이션 통합
1. `index.html` 네비게이션에 Blog 메뉴 추가
2. 모든 페이지 헤더 업데이트
3. 사이트맵 생성

### Phase 6: SEO 및 최적화
1. 메타 태그 추가 (Open Graph, Twitter Card)
2. RSS 피드 생성 (선택사항)
3. robots.txt, sitemap.xml

## 구체적 구현 계획 (방안 2 - 순수 정적 방식 기준)

### Phase 1: 데이터 구조 설계
1. `data/blog-posts.json` - 포스트 메타데이터
2. `posts/` 디렉토리 - HTML/Markdown 콘텐츠 파일

### Phase 2: 블로그 목록 페이지
1. `blog.html` 생성
   - Bootstrap 카드 레이아웃
   - 포스트 썸네일, 제목, 요약, 날짜
   - 태그 필터 기능

### Phase 3: 블로그 상세 페이지
1. `blog-post.html` 템플릿
   - URL 파라미터로 포스트 ID 전달
   - JavaScript로 JSON 데이터 로드
   - 동적 콘텐츠 렌더링
   - 댓글 섹션 (Utterances/Giscus)

### Phase 4: JavaScript 구현
1. `assets/js/blog.js`
   - JSON 데이터 fetch
   - 포스트 목록 렌더링
   - 필터링/검색 기능
   - 페이지네이션

### Phase 5: 스타일링
1. 기존 Bootstrap 테마에 맞춰 스타일 적용
2. 반응형 디자인 보장
3. 코드 하이라이팅 (Prism.js/Highlight.js)

### Phase 6: 글 작성 워크플로우
1. JSON 파일 수정 (메타데이터 추가)
2. 포스트 콘텐츠 HTML 파일 작성
3. Git commit & push

## 권장 사항 정리

### 간편함 우선 → 방안 2 (순수 정적)
- 구현 시간: 2-3일
- 유지보수: 쉬움
- 비용: 무료
- 글 작성: JSON + HTML 파일 편집

### 사용자 경험 우선 → 방안 3 (Decap CMS)
- 구현 시간: 3-5일
- 유지보수: 보통
- 비용: 무료
- 글 작성: 웹 UI에서 Markdown 작성
